Project Aura User Manual

Complete operational guide, interface descriptions, navigation details, and sample transactions for the offline triage hub.

1. Application Overview

Project Aura is a secure, 100% offline-first IT incident analysis and triage assistant. By integrating local LLMs (Google Gemma-2-2B-it or Phi-3.1-mini-4k) with ChromaDB (vector database), the platform parses historical incident ticket databases, digests corporate manuals, classifies tickets, and synthesizes step-by-step resolution guides safely within your local environment.

2. Main Navigation & Screen Sections

Use the persistent horizontal navigation bar to navigate between the four core pages:

Screen Purpose & Features
Home Dashboard Provides high-level KPIs representing ingestion counts, knowledge base files, active LLM status, search success rates, and compute/latency profiling metrics (AVG, P90, per-incident tasks).
Knowledge Management Allows drag-and-drop or file uploads of support ticket CSV files and technical manuals (PDF/DOCX). Displays active file status catalogs.
Macro Categorization Triggers background LLM tasks to batch-classify historical tickets. Displays volume breakdown graphs and descriptions of operational taxonomies.
Search & Triage The primary interactive workspace. Includes tabs for single-query search & RAG guide compilation, and bulk triage processing using CSV upload.

3. Screen Breakdown & How-to-Use

3.1 Knowledge Management Screen

Upload files to feed knowledge context to the vector database:

  1. Go to Knowledge Management in the nav bar.
  2. To ingest incident logs: Click "Choose File" on the CSV uploader, select your tickets dataset (containing columns: number, short_description, description, comments_and_work_notes, closed_note, assigned_to, assignment_group, and cmdb_ci), then click Ingest Incidents CSV.
  3. To upload documentation: Select a DOCX or PDF file (e.g. active_directory_guide.docx), choose the matching CMDB CI application it applies to, and click Ingest Documentation.
  4. Check processing counts and status in the Active Documents Catalog at the bottom.

3.2 Macro Categorization Screen

Classify ticket descriptions into operational taxonomies:

  1. Go to Macro Categorization in the nav bar.
  2. If there are unclassified incidents, click the Run Categorization button. The background classifier will batch process 15 records at a time.
  3. View progress in real time. Once finished, the dashboard will refresh.
  4. Use filters at the top (Start Date, End Date, CMDB CI) and click Apply Filter to segment the charts and tables. Click Export CSV to save the categorized list.

3.3 Search & Triage (RAG) Screen

Execute single and bulk RAG tasks:

  1. Go to Search & Triage in the nav bar.
  2. Single Query: Enter an issue description on the left (e.g., "User account locked out after three password attempts"), choose the CMDB CI application (e.g., Active Directory), and click Search & Synthesize Guide.
  3. Observe results on the right:
    • Assistant Resolution Guide: Dynamically compiles instructions from matching manuals in formatted Markdown.
    • Citations: Links to downloaded source PDFs or Word documents.
    • Deduplicated Matched Incidents: Displays historical tickets, match percentages, assigned personnel, assignment groups, and closed notes.
  4. Bulk Triage: Switch to the "Bulk Triage CSV" tab, upload a CSV containing description and cmdb_ci columns (like the sample CSVs provided), and review the compiled batch guidelines displayed immediately below.

4. Sample Transaction Flows

Scenario A: Standard Active Directory Password Reset

Query: "How do I unlock account or reset expired password for Active Directory domain?"

CMDB CI: Active Directory

Expected Output: The system identifies manuals matching active directory guides, parses steps (like opening ADUC, right-clicking user accounts, and resetting flags), extracts closed notes from past tickets, and displays the assigned technicians who handled AD lockouts.

Scenario B: Bulk Triage Test Ingestion

Action: Go to the Bulk Triage tab and upload bulk_triage_7.csv.

Expected Output: The screen displays a scrollable card stack detailing seven records. Each card shows the ticket query, matched historical incidents list, RAG resolution summary parsed using local model, and clickable citation links to open local documentation files.

Project Aura
Executive Briefing
01. EXECUTIVE BRIEFING

Executive Summary

Project Aura is a monolithic 100% offline, air-gapped support workstation designed to perform ticket triaging, automated macro categorization, and technical document curation directly on secure local systems.

🔒 Air-Gapped Security
Operates entirely without network dependencies, external API calls, or remote connections. Complies with strict security policies.
⚡ Local CPU Execution
Performs in-process inference and embeddings using lightweight GGUF models optimized for CPU workstation architectures.
📈 Knowledge-Driven RAG
Curates reference manuals and builds persistent vector indices to retrieve solution paths and synthesize guided resolutions.
02. BLUEPRINT

Technical Architecture

A single Python process executes the entire UI and offline models, bypassing Inter-Process Communication (IPC) overhead to maximize CPU throughput.

⚙️ Monolithic FastAPI
Serves frontend templates, REST APIs, and runs background workers in the same thread boundary.
  • PostgreSQL DB: Manages 6 schemas mapping incidents, telemetry, and taxonomy.
  • Persistent ChromaDB: Vector index target folder writing directly to /.chromadb.
  • llama-cpp-python: Coordinates CPU llama GGUF model executions.
03. RATIONALE

Design Rationale & Decisions

Engineering decisions were centered on mitigating constraints of offline workstation deployment environments.

🧠 Silent Memory Fallback
Captures memory OOM faults silently during primary LLM loads. Telemetry creates a warning trace, and the manager immediately mounts the lighter **Phi-3.1** backup model to guarantee system availability.
📁 Chroma Metadata Omissions
ChromaDB does not accept None value inputs. Ingested Word documents (which contain no page numbers) omit the page_number key completely rather than passing a null reference, preventing index crashes.
💾 In-Loop Database Commits
Structured ingestion tasks call transactional commits inside their iteration loops rather than in a batch block at completion. This enables gradual progress tracking metrics on dashboards.
04. FUNCTIONAL MODULES

Workstations Overview

Four core workstations partition system capabilities to match secure IT operator tasks:

1. Knowledge Curator
Accepts technical manuals in `.docx` and `.pdf` formats. Performs SHA-256 validation to avoid duplicates, splits text page-by-page, generates vectors, and manages physical folders (`/pending`, `/processed`, `/failed`).
2. Single Triage & RAG Guidance
Allows searching for technical solutions. Performs similarity distance metric conversions, filters matching tickets by application (CMDB CI), discards low-confidence matches below threshold, and prompts the LLM to synthesize Markdown guides.
3. Bulk Incident Triage
Asynchronously processes bulk lists of queries from uploaded CSV files (supporting up to 20 rows per batch) inside background tasks, updating progress bars and exporting output logs as CSV files.
4. Macro Categorization Engine WIP
Uses the local LLM in a background thread to map unclassified ticket descriptions to high-level taxonomies and extract error codes/messages. Features a dashboard with metrics, volume breakdown charts, and date filters. (Some changes required).
05. INGESTION

Ingestion & Vectorization Flow

A robust, encoding-resilient data ingestion pipeline processes support CSV inputs to feed the local vector store.

📊 Parse & Vectorize Sequence
  • Encoding Cascade: Decodes bytes trying UTF-8 first, falling back to CP1252/Latin-1 to prevent crashes on Excel accented characters.
  • Header Normalization: Maps variations like close_notes to standard schema column closed_note.
  • Date Parsing: Supports formats: DD-MM-YYYY HH:MM:SS AM/PM and ISO/MySQL standard YYYY-MM-DD HH:MM:SS.
06. RETRIEVAL

Search & RAG Synthesis

Aura triages queries by matching incident patterns in vectorized documentation libraries before prompting GGUF models.

🔍 Context Filtering Mechanics
  • CMDB CI Pinning: Restricts vector search queries using metadata filters.
  • L2 Score Conversions: Converts Euclidean distance metrics to confidence percentages.
  • Threshold Discard: Discards reference segments scoring under the minimum confidence threshold to prevent hallucinated guides.
07. TELEMETRY

Grouped Telemetry Logs & Observability

A parent-child telemetry architecture logs transaction flows across threads to audit model performance without remote telemetry sinks.

📊 OpenTelemetry Spans
  • Trace ID Propagation: Passes IDs to background tasks to associate sub-spans with parent API transactions.
  • Grouped Tables: Settings page renders parent rows that toggle to reveal nested sub-span logs.
  • Structured CSV Export: Exports linked logs for easy parsing.

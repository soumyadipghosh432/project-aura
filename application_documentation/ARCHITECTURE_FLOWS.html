Project Aura Architecture & Data Flows

Monolithic 100% Offline AI Analysis Engine Developer Blueprint

1. System Architecture Diagram

Project Aura utilizes a monolithic architecture where FastAPI orchestrates both database access and local model execution inside a single thread/event lifecycle. This avoids inter-process latency and network hops, maximizing CPU efficiency.

2. Incident CSV Ingestion & Vectorization Flow

This flow details how raw CSV logs are ingested, saved to PostgreSQL, and vectorized in ChromaDB in an asynchronous background thread to prevent thread-blocking on the server.

1

Upload Validation

The FastAPI endpoint parses the CSV. If schema header checks pass, rows are mapped to Structured models.

2

Database Sync

Relational ticket fields are written into PostgreSQL. Rows default to PENDING process status.

3

Async Worker Dispatch

Spawns a background thread carrying the parent HTTP trace_id context to execute vectorization steps asynchronously.

3. Single Triage & RAG Synthesis Flow

This flow maps how a search query is filtered by CMDB CI, parsed by ChromaDB using L2 distance scoring, and synthesized by the LLM into step-by-step instructions.

1

Pre-filtering

ChromaDB restricts search vector space using metadata query dicts matching `cmdb_ci` variables.

2

Threshold Filtering

L2 distances are parsed. Any manual chunks scoring below threshold variables are omitted from LLM context.

3

RAG Compilation

LLM compiles resolution guide markdown templates and citation files dynamically from surviving chunks.
